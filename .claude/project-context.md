# Контекст проекта Weblate

## Технологический стек

### Основные технологии
- **Astro** `5.13.7` - статический site generator, основной фреймворк
- **Tailwind CSS** `4.1.8` - utility-first CSS фреймворк (используется новая версия v4 с Vite плагином)
- **TypeScript** - с путями импорта через aliases
- **Anime.js** `4.2.2` - библиотека для анимаций

### Дополнительные инструменты
- **@astrojs/sitemap** - генерация sitemap для SEO
- **Prettier** с плагином для Astro - форматирование кода

## Архитектурные решения

### ❌ Что НЕ используется в проекте
- **React, Vue, Svelte, Solid.js** и другие JS фреймворки - проект на чистом Astro
- **UI библиотеки** (Material UI, Chakra UI, shadcn/ui, Ant Design и т.д.)
- **CSS-in-JS** библиотеки (styled-components, emotion)
- **Внешние CDN для шрифтов** (Google Fonts и т.д.) - используются только локальные шрифты

### ✅ Принципы разработки

#### 1. Производительность (Performance)
- Локальные шрифты в формате WOFF2 с preload
- Минимальный JavaScript на клиенте
- Отложенная загрузка аналитики (`defer`)
- Оптимизация изображений (WebP)
- Статическая генерация страниц

#### 2. SEO оптимизация
- Полная структура meta-тегов (Open Graph, Twitter Cards)
- Канонические URL
- XML Sitemap
- Semantic HTML
- Robots meta управление (через env переменные)

#### 3. Доступность (a11y)
- ARIA атрибуты (aria-expanded, inert)
- Управление фокусом
- Навигация с клавиатуры (Escape для закрытия меню)
- Semantic HTML разметка
- Focus trap в модальных окнах

#### 4. Масштабируемость кода
- TypeScript с алиасами путей (`@components/*`, `@styles/*`, etc.)
- Модульная структура компонентов
- Разделение данных в JSON файлах
- Переиспользуемые UI компоненты

## Структура проекта

```
src/
├── assets/
│   ├── fonts/           # Локальные шрифты (WOFF2)
│   └── images/          # Изображения и логотипы
├── components/
│   ├── UI/              # Переиспользуемые UI компоненты
│   └── sections/        # Секции страниц
├── data/                # JSON данные
├── layouts/             # Layout компоненты
├── pages/               # Роуты приложения
├── scripts/             # JavaScript для анимаций
└── styles/              # Глобальные стили
```

## Рекомендации для AI ассистента

### При добавлении новых фич:
1. ✅ Используй нативные Astro компоненты
2. ✅ Используй Tailwind CSS для стилизации
3. ✅ Минимизируй клиентский JavaScript
4. ✅ Следуй существующей структуре алиасов
5. ✅ Обеспечивай accessibility
6. ✅ Оптимизируй для SEO и производительности

### Чего избегать:
1. ❌ НЕ предлагать React, Vue или другие фреймворки
2. ❌ НЕ предлагать UI библиотеки компонентов
3. ❌ НЕ добавлять тяжелые зависимости без необходимости
4. ❌ НЕ использовать внешние CDN (предпочитать локальные ресурсы)
5. ❌ НЕ создавать излишне сложные абстракции

### Приоритеты при решениях:
1. **Производительность** - легкость и скорость
2. **SEO** - видимость в поисковиках
3. **Accessibility** - доступность для всех
4. **Масштабируемость** - чистый и поддерживаемый код
5. **Простота** - минимальная сложность

## Версии инструментов разработки
- Node.js: рекомендуется последняя LTS версия
- npm: используется как package manager
- Prettier: форматирование кода
